## Project Overview

Open Jam is a Taiwanese digital marketplace for creators. This repository currently contains the `Auth` service — a split-screen authentication UI (login, register, forgot password, reset password) with Traditional Chinese copy.

## Commands

All commands should be run from `src/Auth/`:

```bash
# Run dev server (http://localhost:5169 or https://localhost:7280)
dotnet run

# Build
dotnet build

# Publish (release)
dotnet publish -c Release
```

Docker:
```bash
# Build image (run from src/ directory)
docker build -f Auth/Dockerfile -t open-jam-auth .
```

There are no automated tests in this project.

## Architecture

### Backend (ASP.NET Core 8 MVC)

`Program.cs` is minimal — just MVC + static files, no database or auth middleware yet.

`HomeController` handles both GET (render view) and POST (validate + redirect) for each screen. All POST actions use `[ValidateAntiForgeryToken]` and follow the PRG pattern: success → `RedirectToAction`, failure → `return View(model)` with ModelState errors.

**Routes:**

| Method | Path | Action |
|--------|------|--------|
| GET | `/login` | `Login()` |
| POST | `/login` | `Login(LoginViewModel)` |
| GET | `/login-done` | `LoginDone()` |
| GET | `/register` | `Register()` |
| POST | `/register` | `Register(RegisterViewModel)` |
| GET | `/register-sent` | `RegisterSent()` |
| GET | `/forgot` | `ForgotPassword()` |
| POST | `/forgot` | `ForgotPassword(ForgotPasswordViewModel)` |
| GET | `/forgot-sent` | `ForgotSent()` |
| GET | `/reset` | `ResetPassword()` |
| POST | `/reset` | `ResetPassword(ResetPasswordViewModel)` |
| GET | `/reset-done` | `ResetDone()` |

Email is passed between POST and the subsequent sent/done GET via `TempData["Email"]`.

### ViewModels

Located in `src/Auth/Models/`. Each form screen has a corresponding ViewModel with Data Annotations that drive both server-side validation and client-side jquery-validation unobtrusive:

- `LoginViewModel` — `[Required]`, `[EmailAddress]`
- `RegisterViewModel` — `[Required]`, `[EmailAddress]`, `[MinLength]`, `[MustBeTrue]`
- `ForgotPasswordViewModel` — `[Required]`, `[EmailAddress]`
- `ResetPasswordViewModel` — `[Required]`, `[MinLength]`, `[Compare]`

`MustBeTrueAttribute` is a custom attribute that implements `IClientModelValidator` to emit `data-val-mustbetrue` for the Agree checkbox.

### Frontend

No build step — scripts are served as-is from `wwwroot/js/`. Validation is handled by jquery-validation unobtrusive (reads `data-val-*` attributes generated by `asp-for` Tag Helpers).

**Script load order (`_Layout.cshtml`):**
1. jQuery 3.7.1 (CDN)
2. `lib/jquery-validation/dist/jquery.validate.min.js`
3. `lib/jquery-validation-unobtrusive/jquery.validate.unobtrusive.min.js`
4. `js/validation-setup.js` — must load after unobtrusive (uses `$.validator.unobtrusive.adapters`)
5. `js/auth-core.js` — UI behaviours only

**`validation-setup.js`** defines:
- `window.scorePassword(pw)` — returns `{ level: 0–4, color, text, hint }` (shared with auth-core.js)
- `$.validator.addMethod('mustbetrue', ...)` — checkbox must be checked
- `$.validator.addMethod('minpasswordstrength', ...)` — `scorePassword(value).level >= 2`
- Unobtrusive adapters for both custom methods (maps `data-val-*` attribute → rule)
- `$.validator.setDefaults` — `highlight`/`unhighlight` add/remove `err` class on `.input-shell`

**`auth-core.js`** handles only UI behaviours (validation is jquery-validation's job):
- Password strength meter — visual bars, inserted after `.input-shell` on `input` event
- Password visibility toggle — `[data-pwtoggle]`
- Custom checkbox sync — `[data-checkbox]` visual div syncs to real `<input type="checkbox">` by id
- Legal modal — `[data-legal-open/close/ack]`, `[data-legal-scrim]`
- Resend countdown timer — initialised on `[data-resend-screen]` pages
- Tweaks panel — font/accent switcher, `postMessage` integration
- Prefs persistence — `localStorage` key `ojAuthPrefs`

`AuthApp.js` is an alternate single-page version of the controller (unused).

### Rendering pattern

Razor views are server-rendered HTML. Forms use:
- `<form asp-action="..." method="post" novalidate>` — antiforgery token injected automatically by the Tag Helper
- `<input asp-for="PropertyName">` — generates `name`, `id`, and `data-val-*` attributes from ViewModel Data Annotations
- `<span asp-validation-for="PropertyName" class="field-err">` — populated by jquery-validation (client) or ModelState (server)

**Custom checkbox pattern** (e.g., `Agree`, `Remember`): a visual `.checkbox` div (JS-driven) sits alongside a real `<input asp-for="...">` sized to 1×1 px. JS toggles `.prop('checked')` on the real input so the value submits correctly. `$.validator.setDefaults` uses `highlight`/`unhighlight` to add/remove the `err` class on the nearest `.input-shell`.

**Password strength with validation**: password fields that require minimum strength carry both `asp-for` (generates `data-val-required`, `data-val-minlength`) and a manual `data-val-minpasswordstrength="..."` attribute. Both rules run through jquery-validation.

### CSS design system

`site.css` defines CSS custom properties for the color palette (`--c-violet`, `--c-pink`, `--c-orange`, `--c-lime`, `--c-cyan`, `--c-yellow`), typography (`--june-display`, `--june-font`, `--june-mono`), and spacing/radius tokens. The neobrutalist visual style uses hard offsets (`box-shadow: Xpx Xpx 0 var(--border-strong)`) with hover lift effects. Font family switches via `.font-bricolage` / `.font-unbounded` class on `.auth-shell`.

`.field-err:empty` and `.field-validation-valid` are hidden via CSS so empty validation spans don't take up space.

### Tweaks panel

A developer/preview panel (bottom-right) for switching accent gradient and display font. Activated via `postMessage` with `{ type: '__activate_edit_mode' }` from a parent frame. The panel posts `__edit_mode_available` on load and `__edit_mode_dismissed` on close.
