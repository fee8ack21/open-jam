# 商品 Catalog

本頁定義**創作者端**的商店設定與商品上下架功能。消費者端的購買、付款、訂閱 / 追蹤、以及註冊後回溯歷史，見 [[Order]]。

## 商店設定

- **子網域**：`<creator>.openjam.co`，須檢查保留字與重複（見 [[Quota]]、[[Bootstrap]]，開店流程見 [[Auth]]）。
- **商店名稱 + 描述**。
- **Banner + Logo / 頭像**。

## 商品

### 商品欄位

- 標題、描述
- 封面圖
- 分類（平台固定）+ 標籤（創作者自由）
- 定價（見定價模式）
- 預覽內容（接 [[Storage]] 雙層預覽）
- 檔案（多檔 bundle，見檔案與版本）

### 定價模式

- **固定價**：創作者設定固定售價。
- **免費**：免費商品（仍可憑信箱領取，見 [[Order]]）。
- **折扣碼 / 優惠碼**：創作者可發折扣碼。

### 檔案與版本

- 一個商品可包含**多個檔案（bundle）**。
- 支援**版本更新**：創作者更新檔案後，已購買者可取得新版（接 [[Storage]] 檔案版本）。
- 檔案上傳、大小限制、掃毒、轉碼、安全等見 [[Storage]]。

### 商品狀態機

- `Draft`（草稿，僅 Owner 可見）→ `Published`（上架，公開可購買）↔ `Archived`（下架封存，Owner 主動收起、可重新上架）。
- `Suspended`（遭平台停權，強制下架）：由 Admin 停權 / 解除；解除後回到 `Archived`，由 Owner 自行決定是否重新上架。
- **上架條件**：
  - 須先建立至少一個版本（current version）。
  - 檔案須已 **ready**（[[Storage]] 掃毒 + 轉碼完成）。
  - 必填欄位齊全。
- **下架**：封存後可重新上架；**已售出商品仍保留買家下載權**（呼應 [[Auth]] GDPR 與 [[Storage]] 生命週期策略）。

## 分類與標籤

- **平台固定分類**：由平台定義，供探索頁（market-web）組織與篩選。
- **創作者自由標籤**：創作者自行加註，補充分類粒度。

## 預覽

- 採雙層機制：**公開預覽衍生檔**（試看片段、PDF 前幾頁、預覽圖）vs **付費完整檔**，預覽內容由創作者指定。詳見 [[Storage]]。

## 配額

- MVP **不分方案**：所有商店共用一組**固定額度**（暫不做付費訂閱方案），日後要做方案時再改為依方案查詢，API 與資料模型不變。
- 配額項目：
  - 單檔大小上限
  - 單一商品總量上限
  - 帳號總儲存空間上限
  - 上架商品數上限
- 配額於使用者提交確認（資產 confirm）時檢查並扣量，簽發上傳 signed URL 階段不計配額（見 [[Storage]]）；計量與用量統計見 [[Quota]]。

## 內容檢舉與審核

- **任何人（含未登入）可檢舉**不宜內容。
- 檢舉進入**管理員審核**，成立則**下架商品**或**停權創作者**（接 [[Auth]] 管理員停權）。
- 檢舉與處理動作記錄 Audit Log（見 [[Log]]）。

## Seeder

- Catalog 相關的預建資料（平台固定分類、子網域保留字，以及 dev 用的預設使用者與測試商品）由 [[Bootstrap]] 統一協調與紀錄。

## 技術與架構

- .NET 微服務，整體結構慣例見 [[Develop]]。
- 跨服務關聯：配額統計 [[Quota]]、檔案 [[Storage]]、檢舉停權 [[Auth]]、稽核 [[Log]]、購買與下載授權 [[Order]]。
